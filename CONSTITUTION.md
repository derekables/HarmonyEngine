# Harmony Engine Constitution

## Purpose
This project defines a long-term system for creating and editing music in a browser-based environment.

## Principles
- Speed and responsiveness are prioritized
- Keyboard-first interaction is required for core workflows
- Music data is independent of its visual representation
- Users should not lose work under any circumstance
- Complexity should scale with user experience

## Design Rule
All features must support a single underlying music model that can be represented in multiple ways (tablature, notation, playback, etc.)

## Scope
The project begins as a tablature editor but is designed to evolve into a full music composition platform.
